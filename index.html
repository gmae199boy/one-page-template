Stacks

항상 좋은 기술에 대해 고민하며 신기술을 적극적으로 배우려는 자세로 살아가고 있습니다.
기술 구분 로고 활용 수준 평가
JavaScript Node.js를 쓰기 위해 사용한 언어입니다. 팀 활동을 하면서 React 와 함께 사용하기 시작하며 JavaScript의 이해도를 높혔습니다.
Node.js Node.js로 Express, Fastify등 웹 개발 프레임워크를 구축할 수 있으며, React와 MERN Stack 방식으로 조합하여 개발할 수 있습니다.
Schema를 활용해 Request가 들어왔을 때 먼저 검증을 한 뒤 응답을 하는 방식을 채택하는 등, API의 안정성과 검증을 동시에 가져가도록 구현하고 있습니다.
React Redux로 상태관리를 하며, UI를 컴포넌트 별로 나눠서 개발할 수 있습니다. open API나 오픈소스 등, 활용할 수 있는 것은 적극적으로 활용하며 사용합니다.
Express RESTFul구조로 API를 설계하고 개발할 수 있으며 MongoDB 및 Mongoose를 사용하여 CRUD를 제공할 수 있습니다
Fastify 마찬가지로 RESTful 구조로 API 개발하며, MongoDB와 Mongoose와 연동하여 사용하고 있습니다
Schema로 요청, 응답 제어를 하며 안정성을 더욱 높히고 있습니다.
MongoDB Mongoose를 사용하여 높은 생산성으로 API를 개발하고 있습니다
MongoDB와 죽이 잘 맞는 json 친화적, 그리고 sql의 left join과 비슷한 populate를 고려해 스키마를 설계하고 있습니다.
Flutter 간단한 네이티브 브릿지를 작성할 수 있습니다. GetX, Provider로 상태관리를 하며 Google map으로 맵뷰를 만들 수 있습니다. API 요청 시 커스텀 json parser를 작성하여 사용하며 모든 위젯을 컴포넌트로 분리하여 개발하고 있습니다.
Golang http, TCP/IP 소켓통신을 이용해 서버를 구축합니다. OS에 접근하여 commend를 실행하고, go루틴을 사용할 때는 뮤텍스를 써서 메모리 교착상태를 해결합니다.
Docker Docker Container를 활용해 각 어플리케이션을 구동합니다. 여러개의 어플리케이션이 종속적인 구동이 필요하다면 Docker Compose를 이용해 Dockerize화 합니다.
Linux 홈서버를 만들기 위해 사용한 OS입니다. 메모리 관리, CPU 관리와 CRON을 사용하여 일정 관리 등을 하고 있습니다.
객관화하기 어려운 관계로, 제가 이해하고 구현할 수 있는 수준을 작성했습니다.

Front End

React로 프로젝트를 진행하다 최근에는 Flutter로 모바일 앱을 작성 중 입니다.

Back end

Node.js를 가장 많이 사용하며 Express.js, Fastify 등의 프레임워크를 활용하고 있습니다. 최근에는 Golang에 많은 관심을 쏟고 있습니다.

Data base

MongoDB를 사용하고 있으며, Mongoose를 활용하고 있습니다. GraphQL에 관심을 가지게 되었고 프로젝트에 도입을 고려 하고 있습니다.

Utilities

GitHub, Redmine, Notion, Mattermost 등 사용하고 있습니다. 프로젝트를 진행하면서 혼자 사용한다 해도 좋은 툴 들을 찾아가며 사용하는 중 입니다.
